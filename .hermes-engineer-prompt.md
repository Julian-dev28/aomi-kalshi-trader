### The flow
1. User opens `/` → dashboard loads, polls `/api/agent/start` (GET) → sees "Hermes is idle"
2. User clicks [▶ START HERMES] → `POST /api/agent/start` → spawns heartbeat → PID file → button changes to [🛑 STOP]
3. Dashboard polls every 5s:
   - `/api/agent/state` → positions, trades, analyses, config
   - `/api/hl/portfolio` → live mark prices, PnL
   - `/api/agent/start` (GET) → check if heartbeat is alive
4. User clicks [🛑 STOP] → `POST /api/agent/stop` → kills heartbeat → button changes to [▶ START]
5. Everything in one screen. No terminal needed.

## UI REQUIREMENTS

### 1. Top bar
- Logo "hermes-trader" left
- Hermes status badge: green "● ACTIVE" or red "○ IDLE"
- Last cycle timestamp ("3m ago" or "never")
- [▶ START HERMES] button when idle, [🛑 STOP] when active
- UTC clock right side

### 2. Stats row (6 cards in grid)
- EQUITY (from portfolio or agent state)
- 24h PnL (green/red)
- WIN RATE (percentage)
- TOTAL TRADES (win/loss breakdown)
- POSITIONS (count of open)
- SCANNED (markets in last cycle)

### 3. Main content (two columns)
- **Left (25%)**: Live positions panel — each position shows coin, side badge, size, entry, mark, PnL, leverage
- **Right (75%)**: Split into two sections:
  - **Reasoning stream** (scrollable, newest at bottom) — cycle-by-cycle logs
  - **Bottom row**: Recent verdicts (left half) + Trade journal (right half)

### 4. Polling
- Portfolio every 8s
- Agent state every 5s  
- Start status (heartbeat alive check) every 5s
- Trades every 12s

### 5. START/STOP buttons
- START triggers `POST /api/agent/start` → on success, status becomes ACTIVE, button becomes STOP
- STOP triggers `POST /api/agent/stop` → on success, status becomes IDLE, button becomes START
- Both buttons disabled during fetch
- Use simple state machine: `idle → starting → active → stopping → idle`
